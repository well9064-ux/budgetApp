# budgetApp

## 프로젝트 설명
이 저장소는 CSV 기반 가계부 CLI 앱 프로젝트다. 입력된 거래 CSV를 읽고, 거래 추가와 조회, 통계 계산을 CLI에서 다루는 것을 목표로 한다.

## 코딩 규칙
- 모든 함수에는 타입 힌트를 반드시 작성한다.
- 함수 하나의 길이는 최대 50줄을 넘기지 않는다.

## TDD 규칙
- 구현 전에 반드시 테스트를 먼저 작성한다.
- 새 기능은 테스트 실패를 확인한 뒤 구현을 시작한다.

## 품질 규칙
- 순환복잡도는 10 이하로 유지한다.

## 품질 검수 규칙
- 커밋 전에 반드시 `qa_engineer` 서브에이전트로 품질 검수를 진행한다.

## 테스트 실행 명령어
- `pytest`
- `radon cc`

## 커밋 규칙
- 하나의 기능이 개발되면 즉시 커밋하고 푸시한다.
